# ЭкоСледователь (фиджитал веб-игра)

Веб-игра для детей 6–11 лет: ребёнок получает улику → выполняет офлайн-задание дома → отправляет доказательство → взрослый подтверждает в дашборде → ребёнок получает награду.

## Структура
- `frontend/` — React + TypeScript + Vite (Kid/Parent режимы)
- `backend/` — Node.js API (Express) + SQLite (`better-sqlite3`) + загрузка медиа (`multer`)

## Быстрый старт (локально)

### 1) Backend

```bash
cd backend
npm install
npm run dev
```

API поднимется на `http://localhost:5174`, проверка: `GET /api/health`.

### 2) Frontend

```bash
cd frontend
npm install
npm run dev
```

Открой `http://localhost:5173`.

## Как играть
- **Режим ребёнка**: `/kid` → выбрать дело → улика → “Я сделал(а)!” → загрузить фото/аудио/заметку → отправить.
- **Режим родителя**: кнопка “Я родитель” → PIN **1234** + простой пример → дашборд → очередь проверки → подтвердить/отклонить.

## Что уже есть для оценки
- **Core-геймплей**: дела, улики, финальное раскрытие версии дела.
- **Meta-геймплей**: карта леса, альбом улик и стикеров, жетоны, прогресс по делам.
- **Фиджитал-связь**: ребёнок выполняет офлайн-действия дома и отправляет фото/аудио/заметку.
- **Образовательная ценность**: у заданий есть навык, эко-факт и возрастные подсказки.
- **Адаптация 6-11**: режимы 6-8 и 9-11 лет, короткие сессии, озвучка инструкций через Web Speech API.

## Демо-чеклист
1. Открыть `/kid`, выбрать возрастной режим и перейти на карту леса.
2. Открыть дело, посмотреть улики и блок “Чему ты учишься”.
3. Нажать “Послушать задание”, затем отправить доказательство.
4. Открыть родительский режим, пройти PIN и пример, подтвердить/отклонить заявку.
5. Вернуться ребёнком, проверить награду, альбом улик и финальное раскрытие дела.

## Проверка стабильности
Backend:

```bash
cd backend
npm run health
```

Frontend:

```bash
cd frontend
npm run build
```

## Настройки демо
- `VITE_API_BASE` — URL backend API для frontend, по умолчанию `http://localhost:5174`.
- `VITE_PARENT_PIN` — PIN родительского режима, по умолчанию `1234`.

## Примечания
- Валидация офлайн-заданий — **через подтверждение родителем** (без ML/распознавания).
- Медиа хранятся в `backend/uploads/`, записи — в `backend/data.db`.

## Деплой (варианты)
- **Простой демо-вариант**: задеплоить `backend` (Node) на Render/Fly.io, а `frontend` на Vercel/Netlify.\n  В `frontend` задать `VITE_API_BASE` на URL бэкенда.\n- **Один сервер** (если хочется): собрать фронт `npm run build` и раздавать `frontend/dist` из Node (не реализовано здесь — можно добавить при необходимости).
